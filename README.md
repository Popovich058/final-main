# Автоматизация тестирования и публикации Docker‑образа

Проект демонстрирует настройку GitHub Actions workflow для:
* автоматического тестирования кода;
* автоматической публикации Docker‑образа в Docker Hub при добавлении тегов.

## Настройка окружения

Перед запуском workflow убедитесь, что настроены секреты в репозитории GitHub:
* `DOCKERHUB_USERNAME` — имя пользователя Docker Hub;
* `DOCKERHUB_TOKEN` — токен доступа Docker Hub (вместо пароля).

Эти секреты используются в workflow для аутентификации при отправке образа.

## Использование

1. Внесите изменения в код и отправьте их в репозиторий.
2. Проверьте, что job `test-job` успешно выполнился (все тесты пройдены, ошибок нет).
3. Для публикации нового релиза:
   * создайте и отправьте Git‑тег (например, `git tag v1.0.0 && git push origin v1.0.0`);
   * дождитесь завершения `publish-job`.
4. Проверьте результат:
   * образ должен появиться в вашем аккаунте Docker Hub;
   * убедитесь, что образ можно скачать и запустить локально:
     ```bash
     docker pull <ваш-username>/<имя-образа>:<тег>
     docker run -d <ваш-username>/<имя-образа>:<тег>
     ```
